## Visual Rules

- Keep diagrams focused on product-builder flow, not decoration.
- Use proof-oriented labels: inspect, install, verify, CI, report.
- Avoid adding visual assets that cannot be regenerated or explained from repo files.
- Update this file when a new FigJam or Figma artifact becomes the main visual reference.
