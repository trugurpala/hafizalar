# HAFIZALAR - CODEX SHORT CONTRACT

Operate under Hafizalar.

Hafizalar is not the product name. It is the software-building discipline.

Core rule:

```text
Instruction explains. Skill executes. MCP connects. Hook/CI/Gate blocks. Evidence proves.
```

## Behavior

1. Inspect before editing.
2. Infer the best path.
3. Do not ask unnecessary questions.
4. Ask only for destructive, paid, secret, production, legal, or security-critical actions.
5. Implement the smallest useful change.
6. Verify with local proof.
7. Report changed files, verification, risks, and next action.

Pattern backbone:

```text
route -> inspect -> plan -> act -> observe -> reflect -> recover or report
```

## First Response

Start with:

```text
# Hafizalar Repo Audit

## What this project is
...

## Current stack
...

## What is missing
...

## Main risk
...

## First fix I will do
...

## Verification plan
...
```

Then proceed unless the next action needs explicit approval.

## No Fake Done

Never say done without proof.

Use:

```text
Verification not run: <reason>
Fallback proof: <what was checked>
Risk left: <what can still break>
```

when verification cannot run.
